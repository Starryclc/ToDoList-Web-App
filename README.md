# TO DO List

一个纯前端、浏览器本地运行的To-do-list应用，参考 Microsoft To Do的交互思路，支持普通任务、循环任务、任务分组、步骤拆解、本周速览、我的一天、重要标记、搜索、归档、备份、恢复等功能。

## 产品简介

本项目适合个人任务管理场景、不方便安装可执行文件的环境。强调以下目标：

- 打开即用：无需后端、无需登录
- 结构清晰：系统清单 + 自定义清单
- 信息完整：任务标题、备注、步骤、截止日期、循环任务、创建/完成时间
- 长期可用：支持备份恢复与按年归档（导出csv）

## 核心功能

### 1. 清单体系

- 系统清单：
  - `我的一天`
  - `重要`
  - `本周速览`
  - `全部任务`
- 自定义清单：
  - 支持新建、重命名、删除、拖拽排序

### 2. 任务管理

- 新增任务：标题、步骤、截止日期/时间、重复、备注
- 编辑任务：支持修改任务全部字段
- 完成状态切换：支持取消完成
- 重要标记：星标切换
- 添加到我的一天：太阳图标切换

### 3. 步骤子任务

- 支持为任务添加多个步骤
- 步骤可独立勾选、删除
- 主任务完成时，步骤同步完成
- 主任务取消完成时，步骤重置为未完成

### 4. 截止日期与超期规则

- 未完成任务：按当前时间动态判断是否超期
- 已完成任务：按“完成当时”判断是否超期
- 对于仅日期（无具体时间）的任务：当天完成不判定超期
- 超期任务截止日期会标红显示，对于对于仅日期（无具体时间）的任务，虽当天不判定为超期，仍标红以示提醒

### 5. 循环任务

- 支持每天/每周/每月重复
- 当期任务完成后自动创建下一期任务
- 下一期任务步骤自动重置为未完成

### 6. 搜索与展示

- 左侧搜索支持检索：标题、备注、步骤内容
- 搜索结果支持关键词高亮
- 支持“展示详情 / 隐藏详情”
- 支持“展示已完成 / 隐藏已完成”

### 7. 排序能力

- 排序项：
  - 默认排序
  - 按截止日期升序/降序
  - 按创建日期升序/降序
- 当前策略：
  - 未完成任务：
    - 默认排序：重要任务优先，其次按截止日期升序，最后按创建时间倒序兜底
    - 非默认排序：重要任务不再固定置顶，按所选规则参与排序
  - 已完成任务：固定按完成时间倒序（最新完成在最上）

### 8. 本周速览

- 汇总两类任务（周一到周日）：
  - 本周已完成任务（按完成时间）
  - 本周到期但未完成任务（按截止时间）
- 标题下展示本周日期范围
- 不允许手动直接添加到本周速览

### 9. 数据能力（备份/恢复/归档）

- `备份`：导出当前全部数据（JSON）
- `恢复`：导入 JSON 数据
- `归档`：
  - 输入“完成年份”筛选已完成任务
  - 导出为 CSV
  - 可选择导出后从当前任务中移除以减轻存储负担
- 归档 CSV 列顺序：
  - 标题、步骤、备注、所属清单、截止日期、创建时间、完成时间

## 数据存储说明

- 主存储：`IndexedDB`
- 兼容回退：`localStorage`
- 旧版本数据会自动迁移
- 数据为浏览器本地站点数据，清理站点数据后会丢失，建议定期“备份”或“归档”

## 项目结构

- `index.html`：页面结构
- `assets/css/todo-app.css`：样式文件
- `assets/js/todo-app.js`：业务逻辑与数据处理

## 使用方式

1. 直接用浏览器打开 `index.html`，固定保存路径后浏览器收藏即可直接通过浏览器打开
2. 开始管理任务
3. 建议定期点击“备份”导出 JSON
4. 每年或按需使用“归档”导出 CSV 并清理历史完成任务
